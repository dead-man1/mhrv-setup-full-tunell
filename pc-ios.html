مراحل راه‌اندازی

  1. دانلود برنامه

    از صفحه releases سازنده، نسخه مناسب سیستم‌عاملت رو دانلود کن:

    🪟 Windows mhrv-rs-windows-x64.zip 🍎 macOS mhrv-rs-macos-*.zip 🐧 Linux mhrv-rs-linux-*.tar.gz
    💡برای مک ARM (M1/M2/M3): فایلی که aarch64 داره. برای مک قدیمی Intel، فایل x86_64.
  2. اجرای اول و نصب گواهی

    فایل دانلود شده رو از حالت فشرده دربیار. سپس:

    • ویندوز: روی run.bat دابل‌کلیک کن (با Run as Administrator)
    • مک: روی run.command دابل‌کلیک کن
    • لینوکس: در ترمینال ./run.sh
    ⚠️بار اول رمز سیستم رو می‌خواد. این فقط برای نصب یک گواهی محلیست — هیچ‌چیز به سرور خارجی نمی‌ره. گواهی روی سیستم خودت ساخته می‌شه.
    🔒چرا گواهی لازمه؟ چون mhrv-rs ترافیک HTTPS رو local باز می‌کنه، از طریق Apps Script رد می‌کنه، و دوباره رمزگذاری می‌کنه با گواهی خودش. کلید خصوصی این گواهی فقط روی سیستم تو می‌مونه.
  3. راه‌اندازی Apps Script

    اگه VPS اختصاصی داری، به صفحه اصلی برو و کد Apps Script رو از سربرگ «📝 Apps Script» کپی کن. اگه اشتراک از ربات گرفتی، اسکریپت آماده‌ای رو که برات اومده استفاده کن.

    → ساخت Apps Script در صفحه اصلی
  4. دیپلوی Apps Script در گوگل
    • به script.google.com برو
    • New project → کد آماده رو پیست کن
    • کلیک Deploy → New deployment
    • Type: Web app | Execute as: Me | Access: Anyone
    • دکمه Deploy رو بزن
    • Deployment ID رو کپی کن (رشته طولانی)
  5. پر کردن فرم در برنامه

    صفحه برنامه mhrv-rs الان باز هست. این فیلدها رو پر کن:

    • Apps Script ID: همون Deployment ID مرحله قبل
    • Auth key: همون کلید AUTH (رمز VPS)
    • Google IP: پیش‌فرض 216.239.38.120 خوبه
    • Front domain: www.google.com
    • HTTP port: 8085 (پیش‌فرض)
    • SOCKS5 port: 8086 (پیش‌فرض)

    دکمه Save بعد Start. وضعیت سبز شد ✅

    ⚡برای سرعت بیشتر: دکمه scan کنار Google IP رو بزن. سریع‌ترین IP رو پیدا می‌کنه.
  6. تنظیم مرورگر یا سیستم

    الان mhrv-rs روی 127.0.0.1:8085 گوش می‌ده. مرورگر رو وصل کن:

    🦊 فایرفاکس:

    Settings → Network Settings → Manual proxy
    HTTP host: 127.0.0.1, port: 8085
    تیک ✅ Also use this proxy for HTTPS

    🌐 کروم / Edge:

    افزونه Proxy SwitchyOmega نصب کن. سرور: 127.0.0.1:8085

    🍎 مک (کل سیستم):

    System Settings → Network → Wi-Fi → Details → Proxies
    تیک ✅ Web Proxy + Secure Web Proxy، هر دو 127.0.0.1:8085

    🪟 ویندوز (کل سیستم):

    Settings → Network → Proxy → Manual proxy setup
    Address: 127.0.0.1, port: 8085

  7. تست کن!

    یه سایت فیلتر شده باز کن — مثلاً x.com یا youtube.com. اگه باز شد، تمام! 🎉

    ✅دکمه Test در برنامه هم وجود داره — یک‌بار بزن، یه درخواست از طریق رله می‌فرسته و موفقیت/تأخیر رو نشون می‌ده.

🛠 تنظیمات پیشرفته

📡 تلگرام و اپ‌های غیر-HTTP (Upstream SOCKS5)

رله Apps Script فقط HTTP می‌فهمه. برای تلگرام MTProto و اپ‌های غیرمرورگری، باید یه xray محلی هم داشته باشی که به VPS خودت وصل بشه. در config.json:

{ "upstream_socks5": "127.0.0.1:50529" }

این طوری ترافیک HTTP/HTTPS از Apps Script می‌ره، ولی ترافیک TCP خام (تلگرام، SSH، ...) از xray می‌ره.

🔀 چند Deployment برای سرعت بیشتر

هر Deployment ID یه pool 30 درخواست همزمان داره. برای استفاده سنگین، 3-6 Deployment روی حساب‌های گوگل مختلف بساز:

{ "script_id": ["id1", "id2", "id3", "id4", "id5", "id6"], "auth_key": "your-secret" }

🎯 SNI Pool — بهینه‌سازی

به‌طور پیش‌فرض، mhrv-rs بین www.google.com, mail.google.com, drive.google.com, docs.google.com, calendar.google.com می‌چرخه. اگه یکی محلی فیلتر بود:

UI → SNI pool → Test all → Keep ✓ only

یا در config.json:

{ "sni_hosts": [ "www.google.com", "drive.google.com", "docs.google.com" ] }

🌍 Exit Node — برای ChatGPT/Claude/Grok

سرویس‌های پشت Cloudflare (ChatGPT، Claude، Grok) IPهای گوگل رو bot می‌بینن. اگه می‌خوای کار کنن، یه Exit Node کوچک روی Deno Deploy یا Cloudflare Workers راه بنداز.

→ راهنمای کامل Exit Node

🔄 حالت Direct — وقتی Apps Script فیلتر شده

اگه ISP خود script.google.com رو هم بسته، اول mhrv-rs رو در حالت direct اجرا کن — فقط ترافیک گوگل رو تونل می‌کنه و کمکت می‌کنه بری Apps Script رو deploy کنی. بعد بهش به حالت apps_script برگرد.

{ "mode": "direct", "google_ip": "216.239.38.120", "front_domain": "www.google.com" }

روش ۱: اشتراک از Android (پیشنهاد ✨)

این روش راحت‌ترین و بهترینه. یه گوشی Android که اپ mhrv-rs روش هست، تونل رو از طریق hotspot به آیفون می‌ده.

  1. روی گوشی Android اپ رو نصب و راه بنداز

    راهنمای Android رو در سربرگ «📱 اندروید» صفحه اصلی ببین. مطمئن شو سبز شده و کار می‌کنه.

  2. روی Android: تنظیم listen_host

    برو در تنظیمات Advanced اپ، listen address رو از 127.0.0.1 به 0.0.0.0 تغییر بده. این طوری دستگاه‌های دیگه روی همون Wi-Fi می‌تونن وصل بشن.

  3. روی Android: روشن کردن Hotspot

    Settings → Network → Hotspot & Tethering → روشن کن
    IP پیش‌فرض معمولاً 192.168.43.1 هست

  4. روی آیفون: وصل شدن به Hotspot

    Settings → Wi-Fi → اسم hotspot Android رو انتخاب کن

  5. روی آیفون: تنظیم Proxy

    روی همون شبکه Wi-Fi در iPhone:
    Settings → Wi-Fi → دکمه ⓘ کنار اسم شبکه → پایین صفحه Configure Proxy → Manual

    • Server: 192.168.43.1
    • Port: 8080
    • Authentication: خاموش

    Save بزن، تمام!

  6. تست

    Safari باز کن، یه سایت فیلتر باز کن. اگه باز شد، کار می‌کنه ✅

    ⚠️این روش HTTP/HTTPS رو با proxy می‌فرسته. تلگرام، اینستاگرام، WhatsApp چون اپ خودشون هستن، proxy سیستم رو نادیده می‌گیرن. برای اونا باید روش ۳ یا اپ Shadowrocket استفاده کنی.
💡راه‌حل کامل برای همه اپ‌ها: به‌جای System Proxy، از Shadowrocket (پولی، ~$3) استفاده کن که SOCKS5 رو پشتیبانی می‌کنه و کل ترافیک رو routing می‌کنه. تنظیمش رو در روش ۳ ببین.

روش ۲: اشتراک از کامپیوتر

اگه گوشی Android نداری، می‌تونی از PC تونل رو share کنی.

  1. روی PC: راه‌اندازی mhrv-rs

    طبق راهنمای PC (سربرگ بالا، تب «💻 کامپیوتر»)، اپ رو نصب و راه بنداز.

  2. تغییر listen_host به 0.0.0.0

    در UI: تنظیمات → listen host → 0.0.0.0 (به‌جای 127.0.0.1)

    این یعنی هر دستگاه روی همون شبکه می‌تونه به PC وصل بشه.

  3. پیدا کردن IP کامپیوتر
    • ویندوز: Command Prompt → ipconfig → IPv4 Address
    • مک: System Settings → Network → IP
    • لینوکس: ip a

    معمولاً چیزی شبیه 192.168.1.x هست

  4. وصل کردن آیفون به همون Wi-Fi

    آیفون و PC باید روی یه شبکه Wi-Fi باشن (مثلاً مودم خونه)

  5. تنظیم Proxy روی آیفون

    Settings → Wi-Fi → ⓘ شبکه → Configure Proxy → Manual:

    • Server: IP کامپیوتر (مثلاً 192.168.1.50)
    • Port: 8085

    Save

⚠️محدودیت‌ها: اگه PC خاموش بشه، آیفون بدون اینترنت می‌مونه. اگه از خونه بری بیرون، تنظیمات رو باید خاموش کنی.

روش ۳: Shadowrocket مستقیم

این روش ساده‌تره ولی فقط یه قسمت از قابلیت‌های mhrv-rs رو می‌ده — نه Full Tunnel کامل.

  1. خرید و نصب Shadowrocket

    اپ Shadowrocket از App Store (نزدیک $3). نیاز به اپل آیدی خارج از ایران داره.

    → Shadowrocket در App Store
  2. اضافه کردن سرور

    اگه از PC یا Android تونل رو share می‌کنی، در Shadowrocket یه سرور دستی اضافه کن:

    • Type: SOCKS5 (یا HTTP)
    • Address: IP دستگاه دیگه (PC یا Android)
    • Port: 8086 (SOCKS5) یا 8085 (HTTP)
  3. روشن کردن

    دکمه روشن/خاموش بالای Shadowrocket. آیکون VPN در نوار بالا روشن می‌شه ✅

    الان تمام اپ‌ها (Safari, Telegram, Instagram, WhatsApp) از تونل می‌رن.
