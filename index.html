⚠️ هشدار مهم — Gmail

از Gmail اصلی خود استفاده نکنید. برای هر مرورگر (Brave، Firefox، Chrome) یک Gmail جدید و جداگانه بسازید. هر Gmail یک Deployment ID جداگانه می‌دهد. چند deployment روی یک Gmail سرعت اضافه نمی‌دهد.

🚀 مراحل نصب

۱

نصب tunnel-node روی VPS

فایل install.sh را باز کنید و فقط این دو مقدار را تغییر دهید:

TUNNEL_AUTH_KEY="YOUR_SECRET_KEY" ← رمز قوی دلخواه SSH_PORT=22 ← پورت SSH سرور

سپس در Termius اجرا کنید:

nohup bash install.sh > /root/install.log 2>&1 & tail -f /root/install.log

در پایان آدرسی مثل http://1.2.3.4:8080 نمایش داده می‌شود — یادداشت کنید.

۲

راه‌اندازی Apps Script (برای هر Gmail تکرار کنید)

در هر مرورگر با Gmail جداگانه به script.google.com بروید.

New project → کد پیش‌فرض را پاک کنید → فایل CodeFull.gs را Paste کنید.

⚠️ باگ کپی‌پیست: بعد از Paste، خط آخر را چک کنید — اگر خط خالی یا کاراکتر اضافه بود حذف کنید. این باگ رایج باعث خطای Syntax هنگام Deploy می‌شود.

سه مقدار بالای فایل را تغییر دهید:

AUTH_KEY = "YOUR_SECRET_KEY" TUNNEL_SERVER_URL = "http://YOUR_VPS_IP:8080" TUNNEL_AUTH_KEY = "YOUR_SECRET_KEY"

Deploy → New deployment → Web app

Execute as: Me | Who has access: Anyone

اخطار Google: Advanced → Go to [نام پروژه] (unsafe) → Allow — این یک بار برای هر Gmail نیاز است و کاملاً طبیعی است.

Deployment ID (شروع با AKfyc...) را کپی و نگه دارید.

۳

تنظیم اپ mhrv-rs

Mode: گزینه Full tunnel (no cert) را انتخاب کنید.

Deployment IDها: در بخش Apps Script relay روی + Add بزنید و فقط Deployment ID را وارد کنید (نه URL کامل — اپ خودش URL می‌سازد). برای هر Gmail یک ID جداگانه اضافه کنید.

Auth Key: همان رمزی که در CodeFull.gs گذاشتید.

Connection mode: VPN (TUN) — routes every app

بهینه‌سازی SNI: در بخش SNI pool + tester روی Test کنار هر آدرس بزنید. ۲ تا ۳ آدرس با پینگ کمتر (سبزتر) را فعال نگه دارید.
۴

تست VPS

فایل check.sh را در Termius اجرا کنید:

bash check.sh

اگر همه موارد ✓ بود، سیستم آماده است.
