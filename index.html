⚠️ خطای رایج TUNNEL_AUTH_KEY

راهنماهای قدیمی از MHRV_AUTH_KEY استفاده می‌کردند — این اشتباهه! مقدار صحیح TUNNEL_AUTH_KEY است که در install.sh بالا استفاده شده.

✅ سازگاری — کاربران قدیمی

💡 اگه قبلاً CodeFull.gs رو deploy کردی، نیازی به redeploy نیست. همه نسخه‌های v1.7 به بعد با همین اسکریپت کار می‌کنن.
فقط اپ اندروید رو آپدیت کن و کد VPS رو یه بار دوباره بزن.
  • ✅CodeFull.gs (157 خط، req.k) — سازگار با v1.9.x
  • ⚠️Code.gs ساده — فقط HTTP relay کار می‌کنه، Full Tunnel نه
  • ❌نسخه‌های قدیم با req.auth — نیاز به redeploy دارن

⚡ بهینه‌سازی سرعت

  • 💡3-6 Deployment از اکانت‌های مختلف گوگل — اپ خودکار round-robin می‌کنه
  • 💡SNI pool → Test all → فقط ✅ ها — معمولاً accounts.google.com بهترینه
  • 💡Auto-detect Google IP — هر بار اجرا کن
  • 💡App splitting — اپ‌های بدون نیاز به VPN رو exclude کن
  • 💡Advanced → parallel_relay روی 2-3 اگه اتصال ناپایداره
  • 💡Advanced → coalesce (v1.9+) — batch هوشمند، RTT 6s→3s
  • 💡request_timeout_secs: ISP کند → 45-60 | fail-fast → 15

  • 🔴
    Syntax Error در deployخط آخر اضافه رو پاک کن — این یه باگ رایج paste در مرورگر موبایله
  • 🔴
    VPS قفل شد — SSH نمیشهپورت SSH رو قبل از ufw enable باز کن. اگه قفل شد OS reinstall تنها راهه
  • 🔴
    Docker نصب نشد (dpkg error)dpkg --configure -a بزن، بعد: apt install -y docker.io
  • 🟡
    unauthorized در لاگAUTH_KEY در VPS و Apps Script متفاوته — باید دقیقاً یکی باشن
  • 🟡
    TUNNEL_AUTH_KEY vs MHRV_AUTH_KEYمقدار صحیح فقط TUNNEL_AUTH_KEY است — راهنماهای قدیمی اشتباه بودن
  • 🟡
    اتصال قطع و وصلdeployment معیوب خودکار 2 دقیقه blacklist میشه — طبیعیه (v1.8.5+)
  • 🟡
    تلگرام ناپایدارتلگرام MTProto استفاده می‌کنه — از SOCKS5 Proxy استفاده کن: 127.0.0.1:8086
  • 🟡
    SNI همه fail میشنAuto-detect google_ip بزن — شبکه‌ات Google Edge رو بلاک کرده
  • 🟢
    App not installed هنگام آپدیتنسخه قدیم با کلید متفاوت امضا شده — Settings → Apps → mhrv-rs → Uninstall → نصب مجدد
